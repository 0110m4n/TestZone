Module LFI

3. Protection et prévention du LFI

Validation des entrées

La première ligne de défense consiste à valider et filtrer toutes les entrées utilisateur :

// Exemple de validation en PHP
$allowed_files = ['home', 'about', 'contact'];
$page = $_GET['page'];

if (in_array($page, $allowed_files)) {
    include($page . '.php');
} else {
    include('error.php');
}

Techniques de protection

Plusieurs mesures peuvent être mises en place pour prévenir le LFI :

  • Whitelist : N'autoriser que des fichiers spécifiques
  • Basename : Utiliser basename() pour supprimer les chemins
  • Realpath : Résoudre les chemins absolus
  • Open_basedir : Restreindre l'accès aux répertoires
  • Disable functions : Désactiver les fonctions dangereuses

Configuration serveur

La configuration du serveur web peut également aider à prévenir le LFI :

; Configuration PHP
open_basedir = /var/www/html
allow_url_include = Off
allow_url_fopen = Off

; Désactiver les fonctions dangereuses
disable_functions = exec,passthru,shell_exec,system
