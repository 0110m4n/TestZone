1. Introduction au Cross-Site Scripting

Qu'est-ce que le XSS ?

Le Cross-Site Scripting (XSS) est une vulnérabilité de sécurité web qui permet à un attaquant d'injecter du code malveillant (généralement JavaScript) dans des pages web consultées par d'autres utilisateurs. Cette vulnérabilité survient lorsqu'une application web inclut des données non fiables dans une page web sans validation ou encodage approprié.

Le XSS est l'une des vulnérabilités les plus courantes dans les applications web et figure régulièrement dans le top 10 de l'OWASP (Open Web Application Security Project).

Pourquoi le XSS est-il dangereux ?

Les attaques XSS peuvent permettre à un attaquant de :

  • Voler les cookies de session et usurper l'identité d'un utilisateur
  • Rediriger l'utilisateur vers des sites malveillants
  • Modifier le contenu de la page web
  • Enregistrer les frappes clavier (keylogging)
  • Exécuter des actions au nom de l'utilisateur
  • Accéder aux données sensibles stockées dans le navigateur

Exemple simple de XSS

Voici un exemple basique d'une vulnérabilité XSS :

<!-- Page web vulnérable -->
<p>Bonjour <?php echo $_GET['nom']; ?></p>

Si un utilisateur visite l'URL suivante :

http://example.com/page.php?nom=<script>alert('XSS')</script>

Le code JavaScript sera exécuté dans le navigateur de l'utilisateur, affichant une popup avec "XSS".

Attention : Ceci est un exemple à des fins éducatives uniquement. N'utilisez jamais ces techniques sur des sites web sans autorisation explicite.

Contexte historique

Le terme "Cross-Site Scripting" a été inventé par Microsoft en janvier 2000. Initialement appelé "Cross-Site Scripting", l'acronyme CSS était déjà utilisé pour "Cascading Style Sheets", d'où l'adoption de "XSS".

Les premières vulnérabilités XSS ont été découvertes dans les années 1990, mais elles sont devenues plus préoccupantes avec l'essor des applications web dynamiques et d'AJAX.
