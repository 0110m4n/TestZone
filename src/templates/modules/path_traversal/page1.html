Module Path Traversal

1. Introduction au Path Traversal

Qu'est-ce que le Path Traversal ?

Le Path Traversal, aussi appelé Directory Traversal, est une vulnérabilité qui permet à un attaquant d'accéder à des fichiers et répertoires stockés en dehors du répertoire racine de l'application web. Cette attaque exploite une validation insuffisante des noms de fichiers fournis par l'utilisateur.

L'attaquant peut ainsi lire des fichiers arbitraires sur le serveur, incluant le code de l'application, les données, et les fichiers système critiques.

Exemple d'attaque

Voici un exemple typique de path traversal :

// URL normale
http://example.com/download?file=report.pdf

// Attaque path traversal
http://example.com/download?file=../../../etc/passwd
http://example.com/download?file=..\..\..\..\windows\system32\drivers\etc\hosts

La séquence "../" permet de remonter dans l'arborescence des répertoires pour accéder à des fichiers situés en dehors du répertoire autorisé.

Impact potentiel

Les attaques de path traversal peuvent avoir des conséquences graves :

  • Lecture de fichiers de configuration sensibles
  • Accès aux mots de passe et clés de chiffrement
  • Exposition du code source de l'application
  • Lecture des logs système
  • Accès aux fichiers de base de données
