Module IDOR

1. Introduction aux Insecure Direct Object References

Qu'est-ce que l'IDOR ?

L'Insecure Direct Object Reference (IDOR) est une vulnérabilité qui survient lorsqu'une application expose une référence à un objet interne (comme un fichier, un répertoire, ou une clé de base de données) sans contrôle d'accès approprié. Cela permet à un attaquant d'accéder à des données non autorisées en manipulant simplement les paramètres de la requête.

Exemple d'IDOR

Voici un exemple typique d'IDOR dans une URL :

// URL légitime
https://bank.com/account?id=12345

// Attaque IDOR
https://bank.com/account?id=12346
https://bank.com/account?id=12347

Si l'application ne vérifie pas que l'utilisateur a le droit d'accéder au compte 12346, l'attaquant pourra voir les informations d'un autre utilisateur.

Impact de l'IDOR

Les vulnérabilités IDOR peuvent avoir des conséquences graves :

  • Accès non autorisé aux données personnelles
  • Violation de la confidentialité
  • Modification ou suppression de données
  • Escalade de privilèges
  • Conformité réglementaire compromise (RGPD, etc.)
