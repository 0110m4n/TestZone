Module IDOR

2. Identification des vulnérabilités IDOR

Méthodologie de test

Pour identifier les vulnérabilités IDOR, suivez cette approche systématique :

  • Identifier tous les paramètres qui référencent des objets
  • Créer plusieurs comptes de test
  • Tenter d'accéder aux ressources d'autres utilisateurs
  • Tester différents formats d'identifiants
  • Vérifier les contrôles d'autorisation

Techniques de test

Voici les principales techniques pour tester les IDOR :

// Test d'incrémentation séquentielle
/user/profile?id=1001
/user/profile?id=1002
/user/profile?id=1003

// Test avec différents formats
/api/users/123
/api/users/user123
/api/users/123.json

// Test avec encodage
/file?id=MTIz (base64 de "123")
/file?id=%31%32%33 (URL encode)

Outils de test

Plusieurs outils peuvent automatiser la détection d'IDOR :

  • Burp Suite : Intruder et extensions IDOR
  • OWASP ZAP : Scanner automatisé
  • Autorize : Extension Burp pour tests d'autorisation
  • Scripts personnalisés : Python, Bash
